## Key Changes
- **package.json**: Add `"type": "module"`, replace `sapper` with `@sveltejs/kit` + adapter, update scripts
- **Config**: `webpack/rollup.config.js` → `svelte.config.js`, add adapter
- **Files**: `src/template.html` → `src/app.html`, `src/node_modules` → `src/lib`, `src/client.js` → `+layout.svelte`
- **Routes**: `routes/about.svelte` → `routes/about/+page.svelte`, `_layout.svelte` → `+layout.svelte`, `_error.svelte` → `+error.svelte`
- **Data Loading**: `preload()` → `load()` in `+page.js`/`+layout.js`, single `event` arg instead of `page`/`session`, throw `error()`/`redirect()` instead of using `this`
- **Stores**: Import `page`, `navigating` directly from `$app/stores` instead of calling `stores()`
- **Imports**: `@sapper/app` → `$app/navigation`, `$app/stores`; `@sapper/service-worker` → `$service-worker`
- **Links**: `sapper:prefetch` → `data-sveltekit-preload-data`, `sapper:noscroll` → `data-sveltekit-noscroll`
- **Endpoints**: No direct `req`/`res` access; environment-agnostic design
- **HTML Minification**: Use hook with html-minifier package (not included by default)
