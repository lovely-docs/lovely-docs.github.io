Deploy SvelteKit to Cloudflare Workers/Pages with `@sveltejs/adapter-cloudflare`. Install and configure in svelte.config.js with options for platformProxy, fallback (plaintext/spa), and routes (include/exclude patterns, max 100 rules). Workers needs wrangler.jsonc with assets binding. Pages: build command `npm run build`, output `.svelte-kit/cloudflare`. Access Cloudflare bindings via `platform.env` (KV, Durable Objects). Emulated in dev/preview. Use server endpoints instead of `/functions` directory. Static `_headers`/`_redirects` only; dynamic responses via endpoints/hooks. Troubleshoot: enable nodejs_compat flag, reduce worker size by client-side imports, use `read()` from `$app/server` instead of `fs`.
