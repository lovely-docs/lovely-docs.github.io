Build component libraries with `@sveltejs/package`: `src/lib` is public, `svelte-package` generates `dist` with preprocessed components and auto-generated types. Configure `package.json` with `name`, `license`, `files: ["dist"]`, `exports` (with `types` and `svelte` conditions), `svelte` (legacy), and `sideEffects: ["**/*.css"]`. For non-root exports, use `typesVersions` to map types. Avoid SvelteKit-specific modules; use `esm-env` instead. Publish with `npm publish`. All imports must be fully specified with extensions.
