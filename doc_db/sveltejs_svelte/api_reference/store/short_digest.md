**readable/writable** - Create read-only or read-write stores with optional initialization callbacks.

**derived** - Create stores from one or more source stores with aggregation function (sync or async via set/update).

**get** - Synchronously read current store value.

**readonly/fromStore/toStore** - Convert between stores and reactive objects or getter/setter functions.

**Readable/Writable** - Interfaces with `subscribe()` method; Writable adds `set()` and `update()`.
