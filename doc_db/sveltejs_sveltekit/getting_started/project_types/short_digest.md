SvelteKit supports multiple rendering modes and deployment targets: default (SSR + CSR), static site generation with `adapter-static` (or ISR with `adapter-vercel`), single-page apps (CSR only), multi-page apps (with `csr = false` or `data-sveltekit-reload`), separate backends, serverless (`adapter-auto`, `adapter-vercel`, `adapter-netlify`, `adapter-cloudflare`), own server (`adapter-node`), containers, libraries (`@sveltejs/package`), offline/PWA (service workers), mobile (Tauri/Capacitor with `bundleStrategy: 'single'`), desktop (Tauri/Wails/Electron), browser extensions, and embedded devices (use `bundleStrategy: 'single'`).
