Complete Svelte 5 documentation covering all aspects of the framework.

**Getting Started**: SvelteKit (recommended) or Vite setup. `.svelte` file structure with optional `<script>`, `<script module>`, markup, and `<style>` sections. `.svelte.js/.ts` files for reactive logic and state sharing.

**Runes** (Svelte 5 compiler keywords): `$state` for reactive state (plain values stay plain, objects/arrays become deeply reactive proxies); `$state.raw` for non-reactive state, `$state.snapshot` for static snapshots, `$state.eager` for immediate UI updates. `$derived` for computed reactive state with auto-updates; `$derived.by()` for complex derivations. `$effect` for side effects (runs after mount and after state changes, supports teardown functions); `$effect.pre` (before DOM updates), `$effect.tracking()`, `$effect.pending()`, `$effect.root()`. `$props` for component inputs with destructuring, defaults, renaming, rest properties; `$props.id()` generates unique IDs. `$bindable` marks props as two-way bindable. `$inspect` logs reactive value changes (dev-only); `$inspect.trace()` traces what caused re-runs. `$host` accesses custom element host.

**Template Syntax**: Lowercase tags are HTML, capitalized/dot-notation are components. Attributes support unquoted values, JS expressions, or pure expressions. Boolean attributes included if truthy. Shorthand `{name}` for `name={name}`. Spread attributes with `{...things}`. Events via `on` prefix (case-sensitive, delegated for performance). Text expressions in braces; `null`/`undefined` omitted. `{@html string}` injects raw HTML. HTML comments and `<!-- svelte-ignore -->` for warnings. Conditionals: `{#if}...{:else if}...{:else}...{/if}`. Iteration: `{#each items as item}` with optional index, keying `(item.id)`, destructuring, else clause. `{#key expr}` destroys/recreates on change. `{#await promise}pending{:then value}resolved{:catch error}error{/await}`. Snippets: `{#snippet name(params)}...{/snippet}` rendered with `{@render name(args)}`; implicit `children` snippet; optional with `{@render children?.()}`; type with `Snippet<[Types]>`. Directives: `bind:property` for two-way binding (inputs, selects, media, dimensions, contenteditable, components with `$bindable()`); function bindings for validation; `bind:this` for DOM refs. `use:action` or `use:action={params}` mounts actions. `{@attach attachmentFn}` runs reactive function on element. `transition:name` bidirectional (local by default, `|global` for parent changes); `in:` and `out:` unidirectional. `animate:name` on keyed each children for reordering. `style:property` for inline styles with `|important` modifier. `class:name` conditional classes or attribute form. `{@const var = expr}` block-scoped constants. `{@debug var}` logs on changes. Async/await expressions (experimental) in markup with `<svelte:boundary pending={...}>`.

**Styling**: Scoped CSS via hash-based classes (+0-1-0 specificity). `:global(...)` for global styles; `-global-` prefix for global `@keyframes`. CSS custom properties via `--property-name` (static or dynamic), read with `var()`. Nested `<style>` tags bypass scoping.

**Special Elements**: `<svelte:boundary>` error boundaries with `pending` and `failed` snippets, `onerror` handler. `<svelte:window>` bindable: `innerWidth/Height`, `outerWidth/Height`, `scrollX/Y`, `online`, `devicePixelRatio`. `<svelte:document>` bindable: `activeElement`, `fullscreenElement`, `pointerLockElement`, `visibilityState`. `<svelte:body>` for events. `<svelte:head>` inserts into document.head. `<svelte:element this={tagName}>` dynamic elements. `<svelte:options>` per-component compiler options: `runes`, `namespace`, `customElement`, `css`.

**Runtime APIs**: Stores via `svelte/store`: `writable(value, start?)` with `.set()` and `.update()`, `readable(value, start?)` read-only, `derived(stores, fn, initial?)` computed, `readonly(store)` hides write, `get(store)` retrieves value. Context API: `setContext(key, value)`, `getContext(key)`, `hasContext(key)`, `getAllContexts()`, `createContext<T>()` for type-safe pairs. Lifecycle: `onMount(callback)` after DOM mount (can return cleanup), `onDestroy(callback)` before unmount (only hook in SSR), `tick()` resolves after state changes apply. Imperative: `mount(Component, {target, props})` instantiates, `unmount(component)` removes, `render(Component, {props})` server-only HTML, `hydrate(Component, {target})` reuses SSR HTML.

**Advanced**: Testing with Vitest (unit/component), jsdom environment, `@testing-library/svelte`, Storybook, Playwright E2E. TypeScript: `lang="ts"` on script, configure preprocessor for non-type-only features, `tsconfig.json` requirements. Custom elements: `<svelte:options customElement="tag-name">`, options object with `tag`, `shadow`, `props`, `extend`. Svelte 4→5 migration: `let` → `$state()`, `$:` → `$derived()`/`$effect()`, `export let` → `$props()`, `on:` → event attributes, `createEventDispatcher` → callback props, slots → snippets, `new Component()` → `mount()`. FAQ: Interactive tutorial, support via Stack Overflow/Discord, tooling (VS Code extension, prettier-plugin-svelte), routing (SvelteKit), mobile (Tauri/Capacitor).

**API Reference**: Component type, `mount/hydrate/unmount/render`, lifecycle functions, context functions, event handling, state sync (`tick`, `settled`, `flushSync`, `fork`, `untrack`, `getAbortSignal`), animations (flip, transition functions, easing), motion classes (Spring, Tween, prefersReducedMotion), stores, reactive built-ins (MediaQuery, SvelteDate, SvelteMap/Set, SvelteURL), attachments, compiler API (compile, compileModule, parse, preprocess, migrate), server rendering, types (ComponentProps, Snippet, Action).

**Legacy Mode** (Svelte 3/4): Top-level variables auto-reactive, reassignment triggers updates. `$:` reactive statements auto-rerun on dependency changes. `export let` props with optional defaults and renaming. `on:` event handlers with modifiers (preventDefault, stopPropagation, passive, once, self, etc.). `createEventDispatcher()` for custom events. Slots with fallback content, named slots, `let:` for data passing, `$$slots` to check provided slots. `<svelte:component this={ref}>` for dynamic components. Imperative: `new Component({target, props, context})` with `$set(props)`, `$on(event, callback)`, `$destroy()`. Server: `Component.render(props)` returns `{head, html, css}`.
